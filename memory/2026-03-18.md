## Buenas prácticas de ServiceNow (aprendido 2026-03-18)
- **ES5 obligatorio** en todo código server-side (Mozilla Rhino). Prohibido: const/let, arrow functions, template literals, destructuring, spread, async/await, Promise, ES6+.
- **GlideRecord**: Usar `setLimit()`, `getValue()`, evitar queries en bucles (N+1), usar `addQuery()` antes de `query()`.
- **Business Rules**: Antes (validar/modificar), Después (registros relacionados), Async (procesos pesados), Display (carga de formulario). No usar `current.update()` en Before rules.
- **Script Includes**: Patrón `Class.create()`, definir `type`, usar `initialize`, devolver `{success, data/message}`.
- **Client Scripts**: onLoad, onChange, onSubmit, onCellEdit. Usar `g_form` API. Llamadas servidor con `GlideAjax`.
- **Seguridad**: Sanitizar entradas, escapar HTML (`GlideStringUtil.escapeHTML`), verificar roles (`gs.hasRole`), evitar `gs.elevatePrivilege()`.
- **Update Sets**: Un feature/bug por update set, nomenclatura `[TICKET]-descripción`, nunca Default, preview antes de commit.
- **Checklist de revisión**: ES5, Seguridad, Rendimiento, Patrones correctos, Mantenibilidad.
- **Flow Designer**: Preferir para procesos multi-paso, integraciones; Business Rules para validaciones simples.
- **Integraciones REST**: Nunca hardcodear URLs/credenciales, usar System Properties y Credential Aliases, verificar status code, async para no bloquear.
- **ATF**: Tests aislados, datos de prueba prefijados con "TEST-", probar casos negativos.
- **Scheduled Jobs**: Loguear inicio/fin, usar `setLimit()`, procesar en lotes, preferir async sobre polling.
